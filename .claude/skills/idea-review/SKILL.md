# 博士生研究想法评审（Idea Review）

## 适用范围

- **对象**：博士生的早期研究想法草稿（AI/ML 核心方向及交叉领域）
- **输入**：`ideas/` 目录下的文件，或用户直接粘贴的文本片段
- **不适用**：完整基金申请书 → 使用 `nsfc-i2-review`

## 模式判断

根据用户意图选择对应模式：

- **方向检查模式**：用户询问某个研究方向/领域是否适合当前时间点深入，如"这个方向还能做吗"、"XX 领域还合适吗"、"研究方向时效性"。→ 走方向检查流程
- **完整评审模式**：用户提交具体研究想法并请求评审，如"评审我的 idea"、"这个 idea 靠谱吗"。→ 走三关递进评审流程

## 工作流程

### 方向检查模式

1. 读取审查指南：`references/review-prompt.md`（重点阅读「方向检查模式」章节）
2. 明确待检查的研究方向/领域关键词
3. 联网检索领域现状（≥3 轮 WebSearch）：领域综述与 survey、核心质疑与批评、竞争方案与替代路线
4. 输出五段式方向检查报告（领域活跃度 → 核心质疑 → 竞争格局 → 风险判断表 → 总结建议）

### 完整评审模式

1. 读取完整审查指南：`references/review-prompt.md`
2. 接收研究想法（若输入过于简短，先追问关键信息再评审）
3. 按三关递进逻辑审查，每关前用 WebSearch 检索文献对标
4. 输出五段式评审报告（综合判断 → 学术价值 → 风险 → 改进建议 → 文献对标）
