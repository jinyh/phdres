# phdres — 博士研究想法孵化与评审工作台

## 项目用途

本项目是博士生研究想法的管理、评审与迭代平台。用于存储研究想法草稿、对其进行学术评审，以及基金申请书的函评自查。

## 目录结构

```
phdres/
├── ideas/          # 研究想法草稿（每个想法一个 Markdown 文件）
├── .claude/
│   └── skills/
│       └── idea-review/       # Claude 版研究想法评审 skill
└── .agents/
    └── skills/
        └── idea-review/       # Codex 版项目内适配 skill
```

## Skills 使用规则

### idea-review

**触发场景**：支持两种模式——

1. **完整评审**：用户提交研究想法草稿，请求评审、方向判断或可行性分析。典型触发词：评判 idea、想法评审、"这个 idea 靠谱吗"、"帮我看看这个方向"、"值不值得做"。
2. **方向检查**：用户询问某个研究方向/领域是否适合当前时间点深入。典型触发词："这个方向还能做吗"、"XX 领域还合适吗"、"研究方向时效性"、"方向检查"。

**工作流**：
1. 读取 `.claude/skills/idea-review/references/review-prompt.md`，完整加载评审指南
2. **模式判断**：根据用户意图选择「方向检查」或「完整评审」流程
3. **方向检查流程**：联网检索领域现状（≥3 轮）→ 输出五段式方向报告（领域活跃度 → 核心质疑 → 竞争格局 → 风险判断表 → 总结建议）
4. **完整评审流程**：接收想法 → 若输入过于简短先追问 → 三关递进审查（每关前联网检索，总计不少于 8 篇文献）→ 输出五段式评审报告

**适用领域**：AI/ML 核心方向及交叉领域（AI for Science、AI+医疗等）。

## 项目内多代理 Skill 对应关系

- Claude 版 skill：`.claude/skills/idea-review/`
- Codex 版 skill：`.agents/skills/idea-review/`
- 两者评审逻辑和参考资料应保持一致；若流程规则变更，优先更新 Claude 版，再同步到 Codex 版
- Codex 版是项目内适配版本，主要差异仅限于平台措辞，不应单独演化出不同的评审标准

## ideas/ 目录约定

- 每个研究想法独立存为一个 `.md` 文件，命名使用小写+下划线（如 `engram_edit.md`）
- 文件内容建议包含：问题背景、核心方法设想、研究意义、待解决的关键问题
- 评审产出的意见可追加到对应文件末尾，或另存为 `{name}_review.md`

## 注意事项

- idea-review 需要联网检索文献，请确保网络可用
